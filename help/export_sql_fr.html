Exportation vers SQL

Il ne s'agit pas réellement d'un format de fichier, mais cette fonction vous permettra de générer toutes les commandes SQL nécessaires pour importer facilement la liste dans une base de données.

L'idée de départ était de créer une base de données mySQL, consultable à l'aide d'une page écrite en PHP, ce qui serait beaucoup plus interactif qu'une page HTML pour les gens consultant votre liste si vous la placez sur le web.

L'export SQL va créer une table (movies par défaut) contenant les films avec les valeurs des champs choisis. Le numéro du film est automatiquement ajouté car il est la clé primaire de la table. Les numéros des films doivent donc être uniques pour chaque film pour que l'export SQL soit correct. Vous pouvez facilement renuméroter vos films si besoin ou si vous n'êtes pas certain d'avoir un numéro unique pour chaque film.

Si vous exportez les extras des films, une autre table sera créée (extras par défaut) contenant les extras des films avec les valeurs des champs choisis. Le numéro de l'extra et du film associé sont automatiquement ajoutés car ils sont la clé primaire de la table. Les numéros des extras sont toujours uniques donc vous n'avez pas à vous soucier de ceci ici.

Si vous activez l'exportation des images des films, un champ "PICTURENAME" sera ajouté dans la table des films, contenant le nom de fichier de l'image du film.

Si vous activez l'exportation des images des extras, un champ "PICTURENAME" sera ajouté dans la table des extras, contenant le nom de fichier de l'image de l'extra.

Il y a une option pour générer des requêtes UPDATE à la place des requêtes INSERT, si vous voulez mettre à jour les informations sans supprimer au préalable les données existantes. Cela provoque la mise à jour des films sur base de leur numéro, donc cela peut causer des problèmes pour certains films si vous changez leurs numéros. Aussi, si vous exportez les extras, les anciens extras des films mis à jour seront supprimés avant d'insérer les nouveaux.
