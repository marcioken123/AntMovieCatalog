Ces scripts sont de simples fichiers texte, donc vous pouvez les modifier avec n'importe quel éditeur texte. Mais l'utilisation de l'éditeur intégré à Ant Movie Catalog permet de facilement éditer l'en-tête du script via la fenêtre des propriétés.

Le langage utilisé ici un dérivé du Pascal et de Delphi. Si vous n'avez jamais utilisé de Pascal mais que vous connaissez un autre langage (C, C++, PHP, Java ou autre), il ne sera probablement pas trop difficile pour vous de l'apprendre en observant les scripts existants.

Le moteur de script utilisé est Innerfuse Pascal Script de Carlo Kok's, dans sa version 2.

La plupart des fonctions de base sont supportées. Il y a également une série de fonctions et constantes destinées à l'échange de données avec Ant Movie Catalog. Cette page les décrit. N'hésitez pas à parcourir le code des scripts existants pour avoir des exemples réels.

Le script commence par un bloc de commentaires, qui n'est pas visible lorsque le script est ouvert avec l'éditeur de Ant Movie Catalog. Ce bloc contient les différents paramètres modifiables via la fenêtre des propriétés.

Seuls les scripts placés dans le dossier "Scripts" seront affichés automatiquement dans la liste. Ceux-ci doivent avoir une extension ".ifs". Les fichiers avec l'extension ".pas" sont destinés à être inclus par les scripts, afin de fournir des fonctions supplémentaires. Par exemple StringUtils1.pas contient toute une série de fonctions aidant à la manipulation de chaînes de texte.

Pour inclure un de ces fichiers, il suffit de les référencer derrière le mot-clé "uses" dans le tout début du code :

program NewScript;
uses
  fichier1, fichier2;

Dans cet exemple, cela permettra au script d'appeler les fonctions et variables contenues dans fichier1.pas et fichier2.pas.

function CheckVersion(Major, Minor, Revision: Integer): Boolean;
Renvoie True si la version courante du programme est égale au supérieure à la version spécifiée dans les paramètres.

function AcceptLicense(LicenseVersion: Integer): Boolean;
Affiche dans une fenêtre le contenu du champ "Licence" du script tel que défini dans la fenêtre des propriétés avec des boutons "J'accepte" et "Annuler". Lorsque l'utilisateur a accepté la licence, il ne doit plus la réaccepter jusqu'à ce que le numéro de licence passé en paramètre change. Cela permet de réafficher la fenêtre chez tous les utilisateurs lorsque la licence est modifiée, tout en ne les ennuyant pas à chaque exécution du script.

Vide le contenu de la fenêtre - Appelez cette fonction avant d'utiliser la fenêtre car elle n'est jamais vidée automatiquement.

Ajoute un nouvel élément à la fenêtre. Si l'adresse est une chaîne vide ou si la fenêtre est vide, l'élément est ajouté au niveau racine. Sinon il est ajouté en tant qu'élément "enfant". L'adresse est la chaîne retournée par la fenêtre lorsque l'utilisateur sélectionne l'élément.

procedure PickTreeSort;
Tri les éléments de l'arbre.

function PickTreeExec(var Address: string): Boolean;
Affiche la fenêtre. Le paramètre Address contiendra l'adresse correspondant à l'élément sélectionné. La fonction renvoie True si l'utilisateur clique sur OK, False si l'utilisateur clique sur Cancel.

function PickTreeExec3(Message: string; var Address: string): Boolean;
Même que PickTreeExec mais ajoute le paramètre Message pour afficher un message en haut de la fenêtre "PickTree".

function PickTreeExec4(Message: string; var Address: string; var AddressId: Integer): Boolean;
Même que PickTreeExec2 mais ajoute le paramètre Message pour afficher un message en haut de la fenêtre "PickTree".

Functions pour gérer les cookies

procedure AddCookie(cookie: string; hostOrUrl: string);
Ajoute un nouveau cookie ou change sa valeur et ses propriétés pour le host ou l'url spécifiée.
Exemples :
- AddCookie('cok=1; path=/film', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'path=/film' (ce cookie sera uniquement visible pour les url commençant par 'http://www.example.com/film/').
- AddCookie('cok=1; path=/', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'path=/' (ce cookie sera visible pour les url commençant par 'http://www.example.com/').
- AddCookie('cok=1', 'http://www.example.com') : Même que l'exemple précédent, 'path=/' par défaut.
- AddCookie('cok=1; max-age=10', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'max-age=10' pour supprimer ce cookie dans 10 secondes.
- AddCookie('cok=1; expires=Sun, 07-Aug-2016 16:43:12 GMT', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'expires=Sun, 07-Aug-2016 16:43:12 GMT' pour supprimer ce cookie après cette date.
- AddCookie('cok=1; max-age=0', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'max-age=0' pour supprimer ce cookie immédiatement.
- AddCookie('cok=1; expires=Thu, 01-Jan-1970 00:00:01 GMT', 'http://www.example.com') : Ajoute le cookie 'cok=1' pour le host 'www.example.com' avec la propriété 'expires=Thu, 01-Jan-1970 00:00:01 GMT' pour supprimer ce cookie immédiatement.
- AddCookie('cok=', 'http://www.example.com'): Ajoute le cookie 'cok=' pour le host 'www.example.com' pour supprimer ce cookie immédiatement.

procedure SetCookies(cookies: string; hostOrUrl: string);
Ajoute de nouveaux cookies ou change ses valeurs pour le host ou l'url spécifiée (ex: nameCookie1=valueCookieC; nameCookie2=valueCookieD; nameCookie3=valueCookieE).
Cette fonction ne permet pas l'ajout de propriétés pour les cookies (utiliser la fonction AddCookie pour celà). Le 'path' utilisé pour les cookies est '/'.

Fonctions pour extraire les infos de pages web

function GetPage(address: string): string;
Télécharge une page HTML (ou autre fichier texte) via la méthode GET et la retourne sous forme de chaîne.
Retourne vide si une erreur se produit.

function GetPage2(address: string; referer: string): string;
Idem que GetPage, mais permet de donner une adresse de "referer" (la page depuis laquelle on appelle la page suivante). La valeur par défaut est une chaîne vide.

function GetPage3(address: string; referer: string; cookies: string): string;
Idem que GetPage2, mais permet de spécifier des cookies sous forme de chaine de caractères : 'uid=120; pwd=FHNHESW' par exemple (Voir les fonctions pour gérer les cookies pour plus d'options). La valeur par défaut est une chaîne vide.

function GetPage4(address: string; referer: string; cookies: string; content: string): string;
Idem que GetPage3, mais permet de spécifier le content-type sous forme de chaine de caractères ; par défaut les autres fonctions utilisent '*/*' par compatibilité avec les anciennes versions, mais vous pouvez passer '' pour un appel plus standard.

function GetPage5(address: string; referer: string; cookies: string; content: string; headers: string): string;
Idem que GetPage4, mais permet de spécifier des en-têtes additionnels à fournir au serveur. Ils doivent être encodés suivant le format TStrings de Delphi, i.e. une liste d'éléments nom=valeur séparés par des retours à la ligne. La valeur par défaut est une chaîne vide.

function PostPage(address: string; params: string): string;
Télécharge une page HTML (ou autre fichier texte) via la méthode POST et la retourne sous forme de chaîne. Les paramètres doivent être encodés par la fonction URLEncode.
Retourne vide si une erreur se produit.

function PostPage2(address: string; params: string; content: string; referer: string; forceHTTP11: Boolean; forceEncodeParams: Boolean): string;
Idem que PostPage, mais permet de spécifier en plus le content-type (au lieu de 'application/x-www-form-urlencoded' par défaut), donner une adresse de "referer" (la valeur par défaut est une chaîne vide), forcer l'utilisation de HTTP 1.1 (False par défaut pour 1.0) et demander à la fonction d'encoder elle-même les paramètres (False par défaut).

function PostPage3(address: string; params: string; content: string; referer: string; forceHTTP11: Boolean; forceEncodeParams: Boolean; headers: string): string;
Idem que PostPage2, mais permet de spécifier des en-têtes additionnels à fournir au serveur. Ils doivent être encodés suivant le format TStrings de Delphi, i.e. une liste d'éléments nom=valeur séparés par des retours à la ligne. La valeur par défaut est une chaîne vide.

procedure HTMLDecode(var Value: string);
Décode les caractères HTML contenus dans la chaîne "Value" (ex. "&eacute;" -> "é").

function URLEncode(source: string): string;
function URLDecode(source: string): string;
Convertit les caractères en paramètres d'URL (ex. " " -> "%20") et vice versa.

function GetPicture(address: string): Boolean;
Télécharge une image et l'associe au film courant.
Retourne false si une erreur se produit.

function GetPicture2(address: string; referer: string): Boolean;
Idem que GetPicture, mais permet de donner une adresse de "referer" (la page depuis laquelle on appelle la page suivante).

function GetPicture3(address: string; referer: string; content: string): Boolean;
Idem que GetPicture2, mais permet de spécifier le content-type sous forme de chaine de caractères ; par défaut les autres fonctions utilisent '*/*' par compatibilité avec les anciennes versions, mais vous pouvez passer '' pour un appel plus standard.

function GetExtraPicture(extraId: Integer; address: string): Boolean;
Télécharge une image et l'associe à l'extra spécifié du film courant.
Retourne false si une erreur se produit.

function GetExtraPicture2(extraId: Integer; address: string; referer: string): Boolean;
Idem que GetExtraPicture, mais permet de donner une adresse de "referer" (la page depuis laquelle on appelle la page suivante).

procedure RaiseConnectionErrors(raise: Boolean);
Afficher ou non un message d'erreur lorsqu'un problème de connexion se produit durant un GetPage, PostPage, GetPicture, ...

Stocke la valeur dans le champ personnalisé spécifié du film courant.

function ExportPicture(filename: string): Boolean;
Exporter l'image du film courant dans le répertoire spécifié avec le nom spécifié.
Vous pouvez utiliser la fonction PictureExt pour connaître l'extension de l'image.

function ImportPicture(filename: string): Boolean;
Importer une image à partir du fichier spécifié et l'associer au film courant.

function ImportPicture2(filename: string; picImportMethod: integer): Boolean;
Même que ImportPicture mais vous pouvez spécifier comment importer l'image dans le catalogue avec le paramètre 'picImportMethod'.
Le paramètre picImportMethod peut prendre les valeurs suivantes :
- picImportStore (stocker l'image dans le catalogue)
- picImportCopyInCatDir (copier l'image dans le dossier du catalogue)
- picImportCopyInPicDir (copier l'image dans le dossier images du catalogue)
- picImportLinkAbs (seulement lier l'image avec un lien absolu)
- picImportLinkRel (seulement lier l'image avec un lien relatif si possible)

function RemovePicture: Boolean;
Supprimer l'image du film courant.

function CanSetPicture: Boolean;
Similairement à CanSetField, indique si le script peut récupérer une image.

function PictureExists: Boolean;
Indique si le film courant contient une image et si elle existe vraiment (pour une image copiée ou liée).

function GetPictureStatus: Integer;
Retourne l'état de l'image du film courant :
- picStatusUndefined (non défini)
- picStatusNone (pas d'image)
- picStatusStored (l'image est stockée dans la catalogue)
- picStatusCopiedInCatDir (l'image est copiée dans le dossier du catalogue)
- picStatusCopiedInPicDir (l'image est copiée dans le dossier images du catalogue)
- picStatusLinkAbs (l'image est seulement liée par un lien absolue)
- picStatusLinkRel (l'image est seulement liée par un lien relatif)

function GetPictureExt: string;
Retourne l'extension de l'image si le film courant contient une image, '' sinon.

function GetPicturePath: string;
Retourne le chemin du fichier image si le film courant contient une image copiée ou liée, '' sinon.

function GetPictureFullPath: string;
Même que PicturePath mais retourne le chemin absolue.

function GetPictureSize: Double;
Retourne la taille de l'image en octets si le film courant contient une image, -1 sinon.

function GetPictureWidth: Integer;
Retourne la largeur de l'image en pixels si le film courant contient une image, 0 sinon.

function GetPictureHeight: Integer;
Retourne la hauteur de l'image en pixels si le film courant contient une image, 0 sinon.

Fonctions pour manipuler les chaines avec des expressions rationnelles

function RegExprSet(ExprStr: string): Boolean;
Définir l'expression rationnelle ExprStr à utiliser.
Retourne True si l'expression est bien formée, False sinon.

function RegExprExec(InputStr: string): Boolean;
Trouver la première correspondance dans la chaine en entrée InputStr avec l'expression rationnelle.
Retourne True si la chaine définie en entrée vérifie l'expression rationnelle, False sinon.

function RegExprSetExec(ExprStr, InputStr: string): Boolean;
Même que la combinaison des fontions RegExprSet et RegExprExec.
Retourne True si l'expression est bien formée et que la chaine en entrée vérifie l'expression rationnelle, False sinon.

function RegExprExecNext: Boolean;
Trouver la prochaine correspondance dans la chaine en entrée avec l'expression rationnelle.
Retourne True si une correspondance a été trouvée, False sinon.

Trouver une correspondance dans la chaine en entrée à partir de la position spécifiée Pos avec l'expression rationnelle.
Retourne True si une correspondance a été trouvée, False sinon.

function RegExprSubstitute(TemplateStr: string): string;
Retourne la chaine TemplateStr avec '$&' ou '$0' remplacé par l'ensemble des occurrences de l'e.r. et '$n' remplacé par l'occurrence de la sous-expression #n.

  • Si vous voulez placer le caractère '^' ou '\' dans le template, usez le préfixe '\'.
  • Par exemple: '1\$ is $2\\rub\\' -> '1$ is \rub\'.
  • Si vous voulez placer des chiffres après '$n' vous devez délimiter n avec des accolades '{}'.
  • Par exemple: 'a$12bc' -> 'abc', 'a${1}2bc' -> 'a2bc'.

function RegExprSetSubstitute(ExprStr, InputStr, TemplateStr: string): string;
Même que la combinaison des functions RegExprSet, RegExprExec et RegExprSubstitute.
Retourne la chaine TemplateStr avec '$&' ou '$0' remplacé par l'ensemble des occurrences de l'e.r. et '$n' remplacé par l'occurrence de la sous-expression #n.

function RegExprReplace(InputStr, ReplaceStr: string; UseSubstitution: boolean): string;
Retourne la chaine InputStr avec les occurrences de l'e.r. remplacées par la chaine ReplaceStr.
Si UseSubstitution est égale à True, alors la chaine ReplaceStr sera utilisée comme un template pour les méthodes de substitution.

  • Par exemple: Expression := '(block|var)\s*\(\s*(.*)\s*\)\s*';
  • RegExprReplace('BLOCK( test1)', 'def "$1" value "$2"', True); -> 'def "BLOCK" value "test1"'
  • RegExprReplace('BLOCK( test1)', 'def "$1" value "$2"', False); -> 'def "$1" value "$2"'

function RegExprSetReplace(ExprStr, InputStr, ReplaceStr: string; useSubstitution: boolean): string;
Même que la combinaison des functions RegExprSet et RegExprReplace.
Retourne la chaine InputStr avec les occurrences de l'e.r. remplacées par la chaine ReplaceStr.
Si UseSubstitution est égale à True, alors la chaine ReplaceStr sera utilisée comme un template pour les méthodes de substitution.

function SubExprMatchCount: Integer;
Retourne le nombre de sous-expressions trouvées durant la dernière exécution Exec*.
S'il n'y a pas de sous-expr. mais que l'ensemble de l'expr. a été trouvée (Exec* a retouné True), alors SubExprMatchCount=0.
S'il n'y a pas de sous-expr. et que l'ensemble de l'expr. n'a pas été trouvée (Exec* retourne False), alors SubExprMatchCount=-1.
Notez que quelques sous-expr. peuvent ne pas être trouvées et que pour ces dernières RegExprMathPos=RegExprMatchLen=-1 et RegExprMatchStr=''.

  • Par exemple: Expression := '(1)?2(3)?';
  • Exec ('123'): SubExprMatchCount=2, RegExprMatch(0)='123', (1)='1', (2)='3'
  • Exec ('12'): SubExprMatchCount=1, RegExprMatch(0)='12', (1)='1'
  • Exec ('23'): SubExprMatchCount=2, RegExprMatch(0)='23', (1)='', (2)='3'
  • Exec ('2'): SubExprMatchCount=0, RegExprMatch(0)='2'
  • Exec ('7'): Return False: SubExprMatchCount=-1

function RegExprMatch(SubExprIdx: Integer): string;
Retourne la chaine correspondant à la sous-expr. SubExprIdx de la dernière exécution Exec*. (= Copy(InputStr, RegExprMatchPos(SubExprIdx), RegExprMatchLen(SubExprIdx)))
La première sous-expr. à pour valeur SubExprIdx=1, la dernière à pour valeur SubExprIdx=SubExprMatchCount, l'ensemble à pour valeur SubExprIdx=0.
Retourne '' s'il n'y a pas de sous-expr. dans l'e.r. ou que cette sous-expr. n'a pas été trouvée dans la chaine en entrée.

function RegExprMatchPos(SubExprIdx: Integer): Integer;
Retourne la position de la chaine correspondant à la sous-expr. SubExprIdx de la dernière exécution Exec*.
La première sous-expr. à pour valeur SubExprIdx=1, la dernière à pour valeur SubExprIdx=SubExprMatchCount, l'ensemble à pour valeur SubExprIdx=0.
Retourne -1 s'il n'y a pas de sous-expr. dans l'e.r. ou que cette sous-expr. n'a pas été trouvée dans la chaine en entrée.

function RegExprMatchLen(SubExprIdx: Integer): Integer;
Retourne la taille de la chaine correspondant à la sous-expr. SubExprIdx de la dernière exécution Exec*.
La première sous-expr. à pour valeur SubExprIdx=1, la dernière à pour valeur SubExprIdx=SubExprMatchCount, l'ensemble à pour valeur SubExprIdx=0.
Retourne -1 s'il n'y a pas de sous-expr. dans l'e.r. ou que cette sous-expr. n'a pas été trouvée dans la chaine en entrée.

function RegExprModifiers(Modifiers: string): Boolean;
Définir les valeurs par défaut des modificateurs.
Ajoutez les modificateurs 'i','r','s','g','m','x' avant '-' pour les mettre à True and après '-' pour les mettre à False.
Par exemple, la chaine 'g-xi' met le modificateur 'g' à True et les modificateurs 'x', 'i' à False.
Note 1: Tous les modificateurs sont initialisés à True par défaut.
Note 2: Vous pouvez remplacer ces valeurs par défaut en spécifiant les modificateurs dans l'e.r. '(?irsgmx-irsgmx) ...'

  • ModifierI: Insensible à la casse.
  • ModifierR: Utiliser l'e.r. avec une syntaxe étendu pour les russes.
    Si True, alors à-ÿ contient en plus le caractère '¸', À-ß contient e plus le caractère '¨', et à-ß contient tous les caractères russes.
    Vous devez désactiver ce modificateur si il peut interférer avec votre alphabet national.
  • ModifierS: '.' désigne tout type de caractère (sinon est définie comme [^\n]).
  • ModifierG: Désactiver le modificateur G fait basculer tout les opérateurs dans le style non-greedy.
    Donc si le modificateur G = False, alors tout les '*' fonctionnent comme '*?', tout les '+' comme '+?', etc...
  • ModifierM: Traiter la chaine de caractères comme un texte avec plusieurs lignes.
    Autrement dit, il change la signification de `^' et `$' pour prendre en compte le début et la fin d'une ligne n'importe où dans la chaine plutôt que prendre en compte seulement le tout début et la fin de la chaine.
  • ModifierX: Utiliser une syntaxe étendue pour autoriser les commentaires dans l'e.r. (eXtended syntax).
    Permet d'ajouter des commentaires dans l'e.r. comme ceci : 'e.r. # A comment'.

procedure RegExprDebug(Debug: Boolean);
Si Debug = True (False par défaut), alors un message sera affiché lorsqu'une erreur sera détectée.

Classe TStringList

TStringList est une classe qui peut être utilisée pour gérer une liste de chaînes. Elle peut être utile pour la gestion des pages web, étant donné qu'elle permet de facilement accéder aux lignes séparément.

Déclaration : aList: TStringList;
Création : aList := TStringList.Create;
Destruction : aList.Free;

property DelimitedText: string; permet de lire ou écrire la liste complète comme une simple chaîne délimitée par des délimiteurs et des guillemets spécifiques (, et " par défaut) (ex: line1,line2,"line 3","line 4","line5 A, B","line 6 ""A""")
property Delimiter: Char; définit le délimiteur utilisé par DelimitedText (, par défaut)
property QuoteChar: Char; définit le guillemet spécifique utilisé par DelimitedText (" par défaut)

Classe TStringListEx

Même que TStringList avec des méthodes et propriétés supplémentaires.

Déclaration : aList: TStringListEx;
Création : aList := TStringListEx.Create;
Destruction : aList.Free;

Les méthodes et propriétés suivantes sont disponibles :
procedure NaturalSort; trie toute la liste avec un tri naturel (détecte les nombres automatiquement)
property DelimitedCSVText: string; même que DelimitedText mais autorise les chaines avec espaces sans guillemets (ex: line1,line2,line 3,line 4,"line5 A, B","line 6 ""A""")
property DelimitedAMCText: string; même que DelimitedCSVText mais n'utilise pas de guillemets, ignore les délimiteurs entre parenthèses, trim les chaines et ajoute un espace après chaque délimiteur (ex: line1, line2, line 3, line 4, line5 A B, line 6 "A", line 7 (text7 A), line 8 (text8 A, B), line 9 (text9 A, B (C, D)))
Cette propriété devrait aider à formater/trier plus facilement les champs avec des valeurs multiples comme acteurs, genres, ...

Classe TJvSimpleXml et classes liées

Il s'agit d'un parseur XML assez complet, permettant de manipuler des documents XML plus facilement. Toutes ses fonctionnalités ne sont pas accessibles via les scripts, il se peut donc que la liste suivante soit complétée lorsque certaines fonctionnalités soient rajoutées au moteur de script.

La création de l'objet se fait de manière similaire à la TStringList présentée ci-dessus.

Il possède les méthodes et propriétés suivantes :
procedure LoadFromString(Value: string); charge un document XML à partir d'une chaîne
procedure LoadFromFile(FileName: string); charge un document XML à partir d'un fichier
procedure SaveToString: string; stocke le document XML courant dans une chaîne
procedure SaveToFile(FileName: string); stocke le document XML courant dans un fichier
property Root: TJvSimpleXmlElemClassic; élément correspondant au nœud "racine" du document XML

Les différents nœuds du document sont des TJvSimpleXmlElemClassic, dérivant de TJvSimpleXmlElem. Ce dernier possède les méthodes et propriétés suivantes :
property Name: string; nom du nœud : <nom>
property Value: string; valeur du nœud : <nom>valeur</nom>
property Items: TJvSimpleXmlElems; liste des sous-éléments du nœud
property Properties: TJvSimpleXmlProps; liste des propriétés (attributs) du nœud : <nom attr1="val1" attr2="val2">
property Parent: TJvSimpleXmlElem nœud parent
function GetChildIndex(AChild: TJvSimpleXmlElem): Integer; position dans la liste (Items) du nœud "enfant" spécifié
procedure Clear; supprime tous les éléments et propriétés du nœud

La classe TJvSimpleXmlElems possède les méthodes et propriétés suivantes :
property Count: Integer retourne le nombre d'éléments de la liste
function GetItem(Index: Integer): TJvSimpleXmlElem; permet d'accéder à un élément de la liste en spécifiant sa position
function GetItemNamed(Name: string): TJvSimpleXmlElem; permet d'accéder à un élément de la liste en spécifiant son nom
procedure Clear; supprime tous les éléments de la liste
procedure Delete(Index: Integer); supprime l'élément à la position spécifiée
procedure DeleteNamed(Name: string); supprime l'élément ayant le nom spécifié
function Add(Name: string): TJvSimpleXmlElemClassic; crée et ajoute à la liste un élément

Les attributs de chaque nœud sont des TJvSimpleXmlProp, possédant les propriétés suivantes :
property Name: string; nom de l'attribut
property Value: string; valeur de l'attribut

Ceux-ci sont stockés et gérés par la classe TJvSimpleXmlProps, possédant les méthodes et propriétés suivantes :
property Count: Integer; retourne le nombre de propriétés de la liste
function GetItem(Index: Integer): TJvSimpleXmlProp; permet d'accéder à une propriété de la liste en spécifiant sa position
function GetItemNamed(Name: string): TJvSimpleXmlProp; permet d'accéder à une propriété de la liste en spécifiant son nom
procedure Clear; supprime toutes les propriétés de la liste
procedure Delete(Index: Integer); supprime la propriété à la position spécifiée
procedure DeleteNamed(Name: string); supprime la propriété ayant le nom spécifié
function Add(Name: string; Value: string): TJvSimpleXmlProp; crée et ajoute à la liste une propriété

Propriétés des scripts

Le titre, la description et tous les autres attributs/options/paramètres des scripts peuvent être définis via cette fenêtre. Elle est accessible par l'icône "Propriétés" de l'éditeur de scripts.

Le titre du script doit être relativement court ; il s'agit a priori du nom du site duquel les informations sont importées. Le champ "Infos d'Internet" indique si le script sert à importer des informations d'internet. Le champ "Requiert des films" indique si le script requiert un ou plusieurs films du catalogue pour fonctionner. Si le script n'a pas besoin de films, alors aucune sélection de films ne sera demandé avant l'exécution et aucun film ne sera passé au script. Seuls les scripts ayant ces deux derniers champs à "True" sont visibles lorsque la fenêtre de script est appelée via "Extraire des informations" -> "D'Internet via un script". Lorsque la fenêtre est appelée via le menu "Scripting" tous les scripts sont affichés.

Pour chaque option que vous définissez, vous devez définir ses différentes valeurs possibles (nombre entiers positifs, associés à une description).

Outils de débogage

La barre d'outils visible dans l'éditeur de script possède des commandes similaires à celles disponibles dans Delphi :

  • Basculement de point d'arrêt : place un point d'arrêt sur la ligne en cours pour que l'exécution du script soit mise en pause avant exécution de cette ligne. La ligne apparaît avec un fond rouge. Vous pouvez également placer un point d'arrêt en cliquant dans la marge gauche de l'éditeur. Seuls les points d'arrêts placés sur des lignes de code valides seront utilisés. Il n'est malheureusement pas possible d'indiquer visuellement quels sont les point d'arrêts invalides comme le fait Delphi lors de l'exécution d'un programme. Lorsque l'exécution est en pause, vous pouvez avancer à l'instruction suivante manuellement via le bouton "Avancer" ou relancer l'exécution via le bouton "Exécuter".
  • Supprimer les points d'arrêt : supprime en un coup tous les points d'arrêts définis dans le script.
  • Avancer : exécute la ligne en cours (surlignée en bleu) et passe à l'instruction suivante.
  • Exécuter jusqu'au curseur : exécute le script jusqu'à ce qu'il arrive à la ligne courante (génère donc un point d'arrêt "virtuel").
  • Évaluer la variable : affiche le contenu de la variable qui est à la position du curseur.
  • Ajouter un suivi de cette variable : rajoute à la liste de suivis la variable à la position du curseur. Cela permet de connaître à tout moment le contenu de la variable.
